## 2. 核心觀念(務必先讀)

### 觀念 A:兩種操作路徑,做的事一樣
- **手動操作**:你自己在頁面上點按鈕、填欄位。流程透明、看得到每一步。
- **Agent 操作**:到「AI 助手」用中文講需求,Agent 自動呼叫對應功能。適合懶得點、或步驟複雜時。
> 兩者**底層共用同一套引擎與設定**。手動設好的路徑,Agent 也看得到;反之亦然。

### 觀念 B:「報告精靈」vs「圖片對應」差在哪?
這是最多人搞混的地方:

| | 報告精靈 | 圖片對應 |
|---|---|---|
| 目的 | 一份範本 × 一份 Excel → **產出很多份**報告 | 對**單一一份**現成檔做圖片處理 |
| 輸入 | 範本(含 `{{標籤}}`)+ 資料 Excel(每列=一份報告) | 一份 Word/PPT/Excel + 一個照片資料夾 |
| 輸出 | N 份檔案(N = Excel 列數) | 同一份檔(填好圖) |
| 何時用 | 「我要幫 50 個客戶各做一份合約」 | 「我這份測試報告的照片位置要塞圖」 |

### 觀念 C:標籤(Tag)是一次性的工
範本裡的 `{{客戶名稱}}`、`{{店面照}}` 這種**標籤只需要標一次**。同一種格式的報告,標好範本後可以**重複使用無數次**,之後只要換資料 Excel / 換照片資料夾即可。

### 觀念 D:三種「填圖」方式,挑對的用
| 方式 | 適用情境 | 在哪 |
|---|---|---|
| **每列圖片** | 批次產報告時,每份報告配不同照片(照片檔名對 Excel 某欄) | 報告精靈 → Step0 的「每列圖片(選用)」卡 |
| **制式範本(結構化填圖)** | 範本表格/圖片位置**固定**(如測試報告),照片分子資料夾,每夾對一個表 | 圖片對應 → 制式範本模式 |
| **範本共用(靜態圖)** | 一組固定圖(如 logo、封面)依檔名塞進範本的 `{{圖片欄}}` | 圖片對應 → 範本共用模式 |
| **視覺自動** | 報告**沒有任何標籤**,讓 AI 看版面自己決定每張圖放哪 | 圖片對應 → 視覺自動模式 |

## 3. 名詞解釋

- **範本(Template)**:預先排好版、留了 `{{標籤}}` 的 Word/Excel/PPT 檔。產出時標籤會被資料替換。
- **標籤 / Tag**:`{{欄位名}}` 格式的佔位符,例 `{{客戶名稱}}`。文字標籤會被 Excel 對應欄的值取代;圖片標籤會被圖片取代。
- **來源 Excel(資料)**:每一**列**代表要產出的一份報告;每一**欄**對應一個標籤。
- **標題列(Header Row)**:Excel 裡放欄位名稱的那一列(預設第 1 列)。
- **結構化填圖**:不動文件 XML、只置換內部圖片位元組的填圖法 → **圖片絕不跑位**。靠表格/儲存格/標籤等「結構錨點」定位。
- **視覺配圖 / 填字**:沒有標籤時,用多模態 AI「看」渲染後的版面,自己判斷圖片/文字該放哪。
- **Planner 模型**:負責「思考、規劃、文字理解」的文字模型(Agent 對話、AI 建議對應都靠它)。
- **Reviewer 模型**:負責「看圖」的多模態(視覺)模型(視覺配圖、審查都靠它)。

## 4. 前置設定

很多功能要先在 **AI 引擎** 頁設定好模型才能跑。建議第一次使用先做這步。

### 4.1 API Key
- **Gemini**(預設供應商):API key 放在 `AI-/.env` 的 `GEMINI_API_KEY`,**不進版控**。
- **Ollama**(本機模型):在 AI 引擎頁填 `Ollama Endpoint`(如 `http://localhost:11434`)。

### 4.2 選模型(AI 引擎頁)
1. 選**供應商**:Gemini 或 Ollama。
2. 按「測試連線」,確認抓得到模型清單。
3. 設 **Planner 模型**(文字):Agent 對話、AI 建議對應需要。
4. 設 **Reviewer 模型**(視覺):視覺配圖、報告審查需要。

### 4.3 本機 Office(COM)
以下功能需要本機**真的裝了** Word / PowerPoint:
- Word/PPT 渲染預覽、AI 視覺審查、視覺自動配圖(用 pywin32 貼圖)。
- 純文字 / 結構化填圖**不需要** Office(用 python-docx/pptx/openpyxl 直接處理)。

## 5. 手動操作篇

### 5.1 報告精靈(批次產報告)

**用途**:一份範本 + 一份資料 Excel → 批次產出 N 份報告。

頁面右上可切換 **Word / Excel / PPTX** 三種報告類型。流程是 5 步:

```
① 設定來源 → ② 對應標籤 → ③ 驗證 → ④ 產出 → ⑤ 完成/審查
```

#### Step 0 — 設定來源
依報告類型,設定這些欄位(用每欄旁的「選擇」鈕開檔案對話框):

| 欄位 | 說明 |
|---|---|
| 範本 | Word→`.docx`、Excel→`.xlsx`、PPTX→`.pptx`(含 `{{標籤}}`) |
| 資料 Excel | 來源資料,每列=一份報告 |
| 工作表 / 標題列 | 選來源 sheet 與欄位名所在列(預設 1) |
| 輸出資料夾 | 產出檔案要存哪 |
| 檔名規則 | 可用 `{index}` 和欄位名,例:`{客戶}_{index}` → `王小明_1.docx` |
| 圖片寬度(mm) | 僅 Word,嵌圖寬度(預設 80mm) |

設好後下方有**資料預覽卡**,確認讀到的欄位與前幾列正確。

> **「每列圖片(選用)」卡**:展開後可設定「每份報告配不同照片」。
> ① 選**圖片資料夾** → ② 選 **key 欄位**(檔名要對上此欄的值) → ③ 填**圖片欄位**(範本裡的 `{{圖片欄}}` 名稱,如「店面照」) → 按**「比對並寫入」**。
> 系統會把對到的圖片路徑寫進 Excel 的圖片欄,並自動更新資料來源,接著正常往下產出時就會逐列嵌入對應照片。會顯示 `對到 X / 總列數` 與未對到清單。

#### Step 1 — 對應標籤
把 Excel 欄位「貼」到範本標籤的位置。

- **左側**:Excel 欄位調色盤。點一個欄位(如「客戶名稱」)選起來;已對應的欄位顯示綠色 ✓。
- **右側(依類型不同)**:
  - **Word**:選一個段落 → 按「＋ 插入」→ 在該段後插入 `{{客戶名稱}}`。
  - **Excel**:點一個儲存格 → 插入標籤到該格。
  - **PPTX**:只列出範本既有 `{{標籤}}` 供核對;PPT 標籤請在 **PowerPoint 裡手動打** `{{欄位}}`(本工具不直接編 PPT 版面)。
- **(Word 限定)「AI 建議對應」鈕**:讓 AI 一次比對範本內容與 Excel 欄位,自動改名 / 插入標籤,省去手動。需先設好 Planner 模型。
- 進階:在真正的 Word/Excel 視窗按 `Ctrl+Shift+M` 也能即時插標籤(需啟動熱鍵橋接,選用)。

#### Step 2 — 驗證
按「重新檢查」,系統比對範本標籤與 Excel 欄位:
- `missing_in_excel`:範本用到、但 Excel 沒有的欄(會填空)。
- `extra_in_excel`:Excel 有、但範本沒用到的欄(無妨)。
- 全對齊 → ✓ 通過。

#### Step 3 — 產出
按「**開始產出**」。進度即時更新(SSE),右側列出剛產好的檔。可「取消產出」。
產完顯示 `產出 X / 總數`;若有開審查,不合格的會被搬到 `Failed_Reports/`。按「開啟輸出資料夾」直接看結果。

#### Step 4 — 完成/審查
顯示審查狀態。若在 AI 引擎開了審查,這裡會看到評分與分級結果;沒開的話提示可去開。

### 5.2 圖片對應(把圖塞進現成檔)

**用途**:對**單一一份**現成 Word/PPT/Excel 做填圖(不是批次)。上方先設「① 圖片資料夾」。有 3 個模式:

#### 模式 1:制式範本(資料夾 → 各表)— 結構化填圖
適合**測試報告**這種「表格固定、圖片位置固定」的格式;照片分子資料夾(`01`、`02`、`03`…),每個子資料夾對應範本裡的一個表。

| 欄位 | 說明 |
|---|---|
| 制式範本 | `.docx` 或 `.pptx`,已含照片、位置固定 |
| 照片根目錄 | 底下要有 `01`、`02`… 子資料夾,每個=一個樣本 |
| 配對方式 | `auto`(檔名優先+視覺補強,預設)/ `text`(只看檔名)/ `vlm`(只看圖片內容) |
| 輸出檔 | 留空 = 自動命名 |

**流程**:
1. 按 **「① 先預覽對應(dry-run)」** → 不會改檔,只推導「哪張圖對到哪個槽」給你看。會列出每個子資料夾對到幾槽、有沒有未配對的檔。
2. 對應沒問題 → 按 **「② 正式產出」**。圖片用**位元組置換**填入(文件版面一字不動,圖片不跑位)。

> 鐵律:若數量對不上會觸發**安全閘中止**,不會硬塞。先檢查子資料夾結構。

#### 模式 2:範本共用(配 Word/PPT/Excel)— 靜態圖
範本含 `{{圖片欄}}` 標籤,把**一組固定圖**依檔名換進去(例:每份都放同樣的 logo、封面)。

**流程**:
1. 選範本檔(`.docx`/`.pptx`/`.xlsx`),系統自動讀出範本裡的標籤。
2. 按「檔名比對」(用檔名對標籤)或「AI 比對」(語意對應)。
3. 手動微調對應(每張圖 → 選一個欄位,或選「不放」)。
4. 按「套用到範本」。顯示成功數與失敗清單。

#### 模式 3:視覺自動(Word/PPT·無標註)
報告**完全沒有標籤**,讓 AI 看版面自己決定每張圖放第幾頁、緊鄰哪段。

**需要**:先在 AI 引擎選好 Reviewer(視覺)模型;Word 貼圖需本機 Word。
**流程**:選報告檔 + 圖片資料夾 → 按「開始視覺自動配圖」。內部會:渲染每頁成圖 → 把「圖片＋版面＋段落文字」一起給視覺模型 → 模型判斷位置 → pywin32 一次貼上。完成後列出每張圖放到第幾頁 / 哪個錨點。

### 5.3 Excel 搬移
**用途**:把來源 Excel 的欄位資料,搬到目標 Excel 的對應欄。

1. **來源**(左卡):選來源 Excel、工作表、標題列。
2. **目標**(右卡):選目標 Excel、工作表、標題列。
3. **欄位對應**(中卡):每個來源欄選一個目標欄,或「不搬」。可按「自動對應同名」一鍵配好同名欄。
4. **寫入模式**:`附加`(寫到既有資料下方)/ `覆寫`(清空標題列以下再寫)/ `全新`(新建目標檔)。
5. 按「開始搬移」。顯示 `寫入 X / 總數` 列。

### 5.4 AI 引擎(模型與審查)
- **供應商與模型**:選 Gemini/Ollama、測試連線、設 Planner(文字)與 Reviewer(視覺)模型。
- **產出審查**:勾「啟用審查」後,批次產出時 Reviewer 會自動評分。可設**抽樣比例**(1~100%)與**評分標準(rubric)**(自己寫準則,如「圖片需清晰、欄位不可空白」)。
- **呼叫預算**:防止 AI 呼叫失控。可設 Planner / Reviewer 呼叫上限、查目前用量、按「重置計數」歸零。
  - **視覺配速 RPM**:0 = 自動冷卻(避免觸發 API 限流)。
  - **VLM 看圖配對方式**:`batched`(整批送,快)/ `describe`(單張送,準;檔名無意義如 `IMG_001` 時較準)。

### 5.5 設定
- **輸出預設**:預設輸出資料夾、Word/Excel 檔名規則。
- **圖片嵌入**:Word 圖片寬(mm)、Excel 圖片寬(px)。
- **熱鍵橋接**:設 `Ctrl+Shift+M` 寫入目標(Word 範本游標 / Excel 選取儲存格)。為進階功能,不啟動也能用滑鼠在「對應標籤」完成。

## 6. Agent 操作篇(自然語言下指令)

到 **AI 助手** 頁,用中文講需求,Agent 會自動呼叫對應功能。**所有手動功能都能用 Agent 驅動。**

### 6.1 怎麼開始
1. 先到 **AI 引擎** 選好 **Planner 模型**(沒選 Agent 無法思考)。要用到視覺/審查再加選 Reviewer 模型。
2. 回 AI 助手,確認狀態顯示「已連線」。
3. 直接打需求,`Ctrl+Enter` 送出。
4. Agent 缺資訊時會**跳檔案對話框**請你選檔 / 資料夾,或**跳問題**請你確認(別怕,照著選即可)。
5. 「新對話」可清空重來。

### 6.2 Agent 怎麼運作(簡述)
Agent 收到訊息後會:查當前設定 → 缺路徑就請你選檔 → 需要決策就問你 → 產出前先驗證 → 執行 → 回報結果。它**不會自動重試失敗**,失敗會告訴你檔案位置與原因。

### 6.3 各任務的提示詞範例

> 直接照著講即可,不必記工具名稱。括號是 Agent 內部會做的事。

**批次產報告**
- 「幫我用這份客戶資料批次產生 Word 報告」
- 「改用 PowerPoint 範本,產出簡報」 / 「產生 Excel 報告」
  (查設定 → 缺檔請你選範本與資料 Excel → 驗證欄位 → 有缺欄先問你 → 批次產出 → 回報份數與位置)

**自動對應標籤(一鍵標註,Word)**
- 「幫我自動對好 Word 和 Excel 的欄位」
  (AI 比對範本與 Excel → 一筆筆請你確認改名/插入 → 套用 → 再驗證)

**每列配圖**
- 「照片資料夾裡的圖,依檔名對應 Excel 的『編號』欄,填到『店面照』欄,然後批次產出」
  (把照片路徑寫進 Excel 對應欄 → 後續產出逐列嵌圖)

**制式報告結構化填圖**
- 「這份範本表格和圖片位置固定,照片分在 01、02… 子資料夾,幫我自動填圖」
  (先 dry-run 推導對應給你看 → 你確認 → 正式填圖,圖片不跑位)
- 想指定配對法:「用檔名對就好」(text)/「用看圖片內容對」(vlm)

**範本共用靜態圖**
- 「把這個資料夾的圖,依檔名塞進範本的圖片標籤」

**無標籤視覺自動配圖 / 填字**
- 「這份 Word 沒有標籤,幫我用照片資料夾自動配圖」
  (渲染每頁 → 視覺模型判斷位置 → 貼上)
- 「這份報告沒有標籤,幫我把 Excel 第 1 列的資料填到對應位置」

**審查報告**
- 「幫我檢查這份報告的品質」
  (渲染成圖 → 視覺模型依 rubric 評分 → 回報 通過/問題/建議)

**Excel 搬移**
- 「把這個 Excel 的欄位搬到那個 Excel,同名的自動對應,用附加模式」

### 6.4 Agent 功能的前置需求
| 任務 | 需要 |
|---|---|
| 批次產出、AI 建議對應 | Planner(文字)模型 |
| 視覺配圖、視覺填字、審查 | Reviewer(視覺)模型 |
| 渲染預覽、Word 視覺貼圖、審查 | 本機 Word / PowerPoint(COM) |
| 任何 AI 功能 | Gemini API key 或 Ollama endpoint |

> **預算**:AI 呼叫會扣 Planner/Reviewer 預算點數,超額會停。可在 AI 引擎調上限或重置。

## 8. 疑難排解

| 症狀 | 可能原因與解法 |
|---|---|
| AI 助手「未連線」/ Agent 沒反應 | 沒選 Planner 模型 → 到 AI 引擎選好;或 API key / Ollama endpoint 沒設 |
| 視覺配圖/審查報錯 | 沒選 Reviewer 模型,或本機沒裝 Word/PowerPoint |
| 產出檔欄位是空的 | 該欄在 Excel 缺(Step2 驗證的 `missing_in_excel`)→ 補資料或拿掉標籤 |
| 結構化填圖中止 | 觸發安全閘(照片數與槽位對不上)→ 檢查子資料夾結構與數量 |
| 圖片配對對錯 | 換配對方式:檔名沒意義用 `vlm`/`describe`;檔名清楚用 `text` |
| AI 回「已達預算上限」 | 到 AI 引擎調高上限或「重置計數」 |
| PPT 標籤插不進去 | PPT 標籤要在 PowerPoint 裡手動打 `{{欄位}}`,本工具只核對 |
| 改了前端看不到變化 | 強制重新整理(Ctrl+F5);開發中記得 `npm run build` |
| 觸發 API 限流(429) | 系統內建冷卻+退避;可在 AI 引擎調低「視覺配速 RPM」 |
